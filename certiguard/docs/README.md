# Docs Index

This folder is intentionally kept small. Use these documents as the canonical set:

- `HOW_TO_TEST.md` — operational runbook (setup, dashboard linkage, harness commands, troubleshooting)
- `DEMO_TEST_METHODOLOGY.md` — presentation flow and final sign-off testing sequence
- `LAYERS.md` — implementation source-of-truth by layer and module
- `ARCHITECTURE.md` — high-level architecture narrative for judges/stakeholders
- `obfuscation_guide.md` — dynamic noise generation and integration notes

If content overlaps across files, prefer:

1. `HOW_TO_TEST.md` for commands,
2. `LAYERS.md` for implementation truth,
3. `DEMO_TEST_METHODOLOGY.md` for live demo storytelling.
